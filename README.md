# Diagram Viewer

Single vanilla HTML file hosting a standalone WebComponent + localstorage display of a stack og SVG or PNG's in an interactive Diagram view.

## Build

1. Golang is the primary runtime for installing tools using `go tool` in v1.26+
2. JS and CSS generated by esbuild tool, with JS src files optionally referencing esm.sh CDN hosted files.
3. JSON loaded into WebComponent, generated from the output of D2Lang CLI.
4. Display data persisted in localstorage for MVP

## Requirements

1. The input is a flat list of JSON describing the hierarchy of images - this may be wrapped in a single top-level metadata object, e.g. `{"images": [{"id": 1}]}`
2. Clicking on areas of an image should move to the target in the hierarchy
3. The sidebar is another way to navigate through the hierarchy

## Multi-instance

See `examples/multi.html` for a side-by-side demo of two `<diagram-viewer>` elements sharing one manifest, with only the right instance owning the URL hash.
