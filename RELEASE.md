# Release Process

## Steps

1. **Bump version** — update the tag reference in `README.md` and any version constants.
2. **Build** — run `task build` to produce `dist/diagram-viewer.js`.
3. **Generate SRI hashes** — run `task release-hashes` to compute and print the `sha384` integrity hash for the built artefact.
4. **Verify CDN match** — `release-hashes` automatically re-fetches the file from jsdelivr after tagging and confirms it matches the local hash. If it fails, do **not** publish the hash.
5. **Tag** — `git tag v<version> && git push --tags`.
6. **Release notes** — include the SRI hash in the GitHub release body so consumers can copy it directly.

## Release Notes Template

```markdown
## v0.X.Y

### SRI Hash

Use the following integrity value when loading from CDN:

    integrity="sha384-<HASH>"

Generated by `task release-hashes` against `dist/diagram-viewer.js`.

### Changes

- …
```

## Generating the Hash Manually

```sh
task release-hashes
```

This prints the base64-encoded sha384 hash suitable for the `integrity` attribute. The same hash should be pasted into the release notes and communicated to consumers.
